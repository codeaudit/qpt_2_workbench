# QPT Spec Lookup

The spec is `QPT_2x_Consolidated_Specification_r2.7.md` at the project root. Answer from the file, not from memory.

## Procedure

1. Read or grep the spec file for the term. Quote the registry (Part IX) or the defining section verbatim where possible.
2. Cite precisely: §7.1, A13, L3, Part VI — never "the spec says somewhere".
3. Use canonical 2.7 glyphs only: `(○) [□] {△}`, `𝒢 = R ⊓ G`, `S = ρ − δ − γ log k`. Deprecated forms (Ω, κ, ν, ρd, dr, ε_*, α±) exist only in the Part X deprecation table — if the user quotes one, map it to the current glyph and say so.
4. Scoped glyphs: `⦿` + category subscript = sign trichotomy; `⦿` + Latin = viable-system loop; bare infix `⊗` = composition; `α`/`ι` inside `⟨⟩` = Abduction/Induction, outside = alignment measure. Disambiguate before answering.

## Hard rules

- If it is not in the document, say "the spec does not define this" — never invent an axiom, number, or section.
- Distinguish the gate's qualitative statement (𝒢 ≡ R ⊓ G, §7) from its quantitative instantiation (S ≥ θ, §8) — most confusions live there.
- The unification claim (§7.4): every gate-like construct is the same operator at another resolution. When a question spans two Parts, check whether this claim is the actual answer.
