# Promotion Paths (the meta-workflow)

Four legal paths, each with a precondition and a transformation. Promotions mutate the card's shape — check the result, don't assume it.

## The four paths

**1. Protocol · closure → Dialectic · explore** (§15)
When: the transformation resolved, but its question is now contested across people or agents.
Becomes: position `⟨⦿△ ≡ ⊣α⟩`, fresh agent handle (next `uₙ`), Γ = 0.70, genesis cleared. Gate fields (trajectory, metrics) are stripped — positions don't carry S.

**2. Dialectic · synthesize → Protocol · initiation** (§7.4)
When: an emergent constructor exists (genesis declared — A7 was required to get here).
Becomes: transformation with default gate metrics (ρ .80, δ .10, γ .25, k 3, θ .40), trajectory recruited→grounded, sign `⟨⦿□ ⇢ ⊣□⟩`. It must pass the gate like anything else — genesis is not automatically living.

**3. Protocol + death mode → Resolution · phase0** (§11/§29)
When: a card is diagnosed fossil, residue, or imposition. Dead structure is not forced through the gate; it is excavated.
Becomes: intervention; the diagnosis travels as tags (`† fossil`, `structural layer`). Sign `⟨⦿□ ⇢ ⊣□⟩`.

**4. Resolution · phase5 → Protocol · initiation** (§29)
When: evolutionary iteration produced a redesigned structure.
Becomes: transformation as in path 2 — the loop returns to living process.

## Preconditions and refusals

- Wrong column, wrong state: there is no path. Say which precondition fails instead of forcing one (e.g. promote from `gate` — move it to `closure` first if it's living).
- Paths not taken manually: the trace records `⇧ promote` with the governing section, and an origin chip records lineage. Verify both after promotion.
- Promotion switches the active board — call `set_board` afterward if continuing work there.

## After promoting

Check: card kind and sign class for the new board, origin line, and what was stripped (metrics? tags? Γ?). Report the transformation in one sentence: what it was, what it became, why the path was legal.
